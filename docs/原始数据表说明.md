# 数据表说明

本文整理以下 4 张表的用途与字段定义：

- `auth.users`
- `catalog.videos`
- `catalog.video_user_states`
- `semantic.coarse_unit`

说明来源：

- `catalog.videos`、`catalog.video_user_states` 的大部分字段说明来自数据库内的表注释和列注释。
- `auth.users` 的表级说明来自数据库注释；多数列级说明基于 Supabase Auth 的标准字段语义和字段名归纳，以下以“推断”标记。
- `semantic.coarse_unit` 当前库里没有列注释，以下说明基于字段名、约束和样本数据归纳，以下以“推断”标记。

## auth.users

表描述：

`auth.users` 是 Supabase Auth 的核心用户表，用于存放登录身份、认证状态、联系信息和部分认证流程中间状态。该表位于受保护的 `auth` schema 下。

关键点：

- 主键：`id`
- 唯一约束：`phone`
- 业务上常被其他表通过 `auth.users.id` 关联
- `is_sso_user = true` 时，邮箱可能允许重复

字段定义：

| 字段                          | 类型          | 可空 | 默认值                    | 说明                                                           |
| ----------------------------- | ------------- | ---- | ------------------------- | -------------------------------------------------------------- |
| `instance_id`                 | `uuid`        | 是   |                           | Auth 实例 ID，通常用于多租户或实例隔离语义（推断）             |
| `id`                          | `uuid`        | 否   |                           | 用户主键                                                       |
| `aud`                         | `varchar`     | 是   |                           | audience，通常表示用户所属认证受众，如 `authenticated`（推断） |
| `role`                        | `varchar`     | 是   |                           | Auth 角色，常见为 `authenticated`（推断）                      |
| `email`                       | `varchar`     | 是   |                           | 用户邮箱（推断）                                               |
| `encrypted_password`          | `varchar`     | 是   |                           | 加密后的密码哈希（推断）                                       |
| `email_confirmed_at`          | `timestamptz` | 是   |                           | 邮箱确认时间（推断）                                           |
| `invited_at`                  | `timestamptz` | 是   |                           | 邀请注册时间（推断）                                           |

| `created_at`                  | `timestamptz` | 是   |                           | 账号创建时间（推断）                                           |
| `updated_at`                  | `timestamptz` | 是   |                           | 账号更新时间（推断）                                           |

关联关系：

- `catalog.videos.upload_user_id -> auth.users.id`

## catalog.videos

表描述：

`catalog.videos` 是视频主表，记录上传者、视频生命周期状态、媒体处理结果和 AI 分析结果。

关键点：

- 主键：`video_id`
- 外键：`upload_user_id -> auth.users.id`
- 约束：`raw_file_size > 0`
- 状态拆成 3 层：总体状态、媒体处理状态、AI 分析状态

字段定义：

| 字段                  | 类型                   | 可空 | 默认值              | 说明                                                                                           |
| --------------------- | ---------------------- | ---- | ------------------- | ---------------------------------------------------------------------------------------------- |
| `video_id`            | `uuid`                 | 否   | `gen_random_uuid()` | 主键：UUID（默认 `gen_random_uuid()`），也可显式传入                                           |
| `upload_user_id`      | `uuid`                 | 否   |                     | 上传者用户 ID，关联 `auth.users.id`，受 RLS 约束                                               |
| `created_at`          | `timestamptz`          | 否   | `now()`             | 记录创建时间                                                                                   |
| `updated_at`          | `timestamptz`          | 否   | `now()`             | 最近更新时间，更新时通常由触发器刷新                                                           |
| `title`               | `text`                 | 否   |                     | 视频标题，必填                                                                                 |
| `description`         | `text`                 | 是   |                     | 视频描述，可选长文本                                                                           |
| `raw_file_reference`  | `text`                 | 否   |                     | 原始对象位置，如 `gs://bucket/path/file.mp4`                                                   |
| `status`              | `catalog.video_status` | 否   | `'pending_upload'`  | 总体状态：`pending_upload -> processing -> ready/published`，也可能 `failed/rejected/archived` |
| `version`             | `bigint`               | 否   | `1`                 | 乐观锁版本号，每次业务更新自增                                                                 |
| `media_status`        | `catalog.stage_status` | 否   | `'pending'`         | 媒体阶段状态，如转码、封面处理                                                                 |
| `analysis_status`     | `catalog.stage_status` | 否   | `'pending'`         | AI 阶段状态，如 ASR、标签、摘要                                                                |
| `media_job_id`        | `text`                 | 是   |                     | 最近一次媒体流水线任务 ID                                                                      |
| `media_emitted_at`    | `timestamptz`          | 是   |                     | 最近一次媒体任务完成时间，用于拒绝旧事件                                                       |
| `analysis_job_id`     | `text`                 | 是   |                     | 最近一次 AI 任务 ID                                                                            |
| `analysis_emitted_at` | `timestamptz`          | 是   |                     | 最近一次 AI 任务完成时间，用于拒绝旧事件                                                       |
| `raw_file_size`       | `bigint`               | 是   |                     | 原始文件大小，单位字节，且必须大于 0                                                           |
| `raw_resolution`      | `text`                 | 是   |                     | 原始分辨率，如 `3840x2160`                                                                     |
| `raw_bitrate`         | `integer`              | 是   |                     | 原始码率，单位 kbps                                                                            |
| `duration_micros`     | `bigint`               | 是   |                     | 视频时长，单位微秒                                                                             |
| `encoded_resolution`  | `text`                 | 是   |                     | 主转码分辨率，如 `1920x1080`                                                                   |
| `encoded_bitrate`     | `integer`              | 是   |                     | 主转码码率，单位 kbps                                                                          |
| `thumbnail_url`       | `text`                 | 是   |                     | 主缩略图 URL 或路径                                                                            |
| `hls_master_playlist` | `text`                 | 是   |                     | HLS 主播放清单 URL 或路径                                                                      |
| `difficulty`          | `text`                 | 是   |                     | AI 评估难度，当前为自由文本                                                                    |
| `summary`             | `text`                 | 是   |                     | AI 生成摘要                                                                                    |
| `tags`                | `text[]`               | 是   |                     | AI 生成标签数组                                                                                |
| `raw_subtitle_url`    | `text`                 | 是   |                     | 原始字幕或 ASR 输出 URL/路径                                                                   |
| `error_message`       | `text`                 | 是   |                     | 最近一次失败或拒绝原因                                                                         |

## catalog.video_user_states

表描述：

`catalog.video_user_states` 记录用户对视频的互动状态，是一个面向查询的投影表，用来汇总 `liked / bookmarked / watched` 等标记。

关键点：

- 一行通常表示一个用户对一个视频的聚合状态
- 当前字段设计偏向读模型，而不是事件明细表
- `occurred_at` 表示来源事件发生时间，`updated_at` 表示投影更新时间

字段定义：

| 字段             | 类型          | 可空 | 默认值  | 说明                           |
| ---------------- | ------------- | ---- | ------- | ------------------------------ |
| `user_id`        | `uuid`        | 否   |         | 用户主键                       |
| `video_id`       | `uuid`        | 否   |         | 视频主键                       |
| `has_liked`      | `boolean`     | 否   | `false` | 是否点赞                       |
| `has_bookmarked` | `boolean`     | 否   | `false` | 是否收藏                       |
| `has_watched`    | `boolean`     | 否   | `false` | 是否已观看                     |
| `occurred_at`    | `timestamptz` | 否   | `now()` | 来源 Engagement 事件的发生时间 |
| `updated_at`     | `timestamptz` | 否   | `now()` | 该状态最后一次更新时间         |

## semantic.coarse_unit

表描述：

`semantic.coarse_unit` 看起来是语义知识库里的“粗粒度语义单元”表。按当前数据样本，它存的是词、短语、语法项这一级的标准化条目，并通过 `fine_unit_ids` 关联到更细粒度的释义或 sense。

关键点：

- 主键：`id`
- `kind` 受检查约束限制，只能是 `word`、`phrase`、`grammar`
- `fine_unit_ids` 用数组方式挂接细粒度语义单元
- `original_defs` 保留原始释义文本数组
- 以下字段说明均为推断

字段定义：

| 字段               | 类型          | 可空 | 默认值                                   | 说明                                                   |
| ------------------ | ------------- | ---- | ---------------------------------------- | ------------------------------------------------------ |
| `id`               | `bigint`      | 否   | `nextval('semantic.coarse_unit_id_seq')` | 主键，自增 ID（推断）                                  |
| `kind`             | `text`        | 否   |                                          | 条目类型，只允许 `word` / `phrase` / `grammar`（推断） |
| `label`            | `text`        | 否   |                                          | 条目主展示文本，如单词或短语本身（推断）               |
| `lang`             | `text`        | 否   | `'en'`                                   | 条目语言，当前样本主要为英文（推断）                   |
| `pos`              | `text`        | 是   |                                          | 词性，如 `noun`、`adjective`（推断）                   |
| `english_def`      | `text`        | 是   |                                          | 英文释义或归一化英文说明（推断）                       |
| `chinese_def`      | `text`        | 是   |                                          | 中文释义（推断）                                       |
| `chinese_criteria` | `text`        | 是   |                                          | 中文判定标准或释义边界描述（推断）                     |
| `chinese_label`    | `text`        | 是   |                                          | 中文短标签（推断）                                     |
| `english_label`    | `text`        | 是   |                                          | 英文短标签（推断）                                     |
| `pattern`          | `jsonb`       | 是   |                                          | 结构化模式信息或匹配规则（推断）                       |
| `status`           | `text`        | 否   | `'active'`                               | 条目状态，如是否启用（推断）                           |
| `created_at`       | `timestamptz` | 否   | `now()`                                  | 创建时间（推断）                                       |
| `updated_at`       | `timestamptz` | 否   | `now()`                                  | 更新时间（推断）                                       |
| `version`          | `integer`     | 否   | `1`                                      | 版本号（推断）                                         |
| `fine_unit_ids`    | `bigint[]`    | 否   |                                          | 关联的细粒度语义单元 ID 列表（推断）                   |
| `original_defs`    | `text[]`      | 否   |                                          | 原始释义文本列表（推断）                               |

样本观察：

- `label = 'hood'`，`kind = word`，`fine_unit_ids = {130409}`
- `label = '14 July'`，`kind = phrase`
- `label = '1 Kings'`，`kind = phrase`

这说明该表更像“词条层”而不是“义项层”；真正更细的语义拆分应在 `semantic.fine_unit` 中。
