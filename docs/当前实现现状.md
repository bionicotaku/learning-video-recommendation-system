## 1. 总体结论

当前仓库已经完成第一轮“项目骨架与基础设施”落地，但仍然停留在**可编译、可迁移、可生成 SQL 访问代码、可并行扩展**的阶段，还没有进入任何 Learning engine 或 Recommendation 的业务逻辑实现阶段。

更准确地说，当前仓库已经具备以下能力：

1. 仓库级 Go 工程基线已建立
2. 仓库内置 DB 工具 `dbtool` 已建立
3. `catalog / learningengine / recommendation` 三个顶层模块的基础目录结构已建立
4. `learningengine` 与 `recommendation` 的 owner migration 已写入仓库
5. `learningengine` 与 `recommendation` 的 query / `sqlc` / mapper / repository / tx 基础层已建立
6. DTO、repository interface、usecase interface 已建立
7. `make check` 与 `make sqlc-generate` 在当前仓库内已验证通过

但当前仓库**还没有**以下内容：

1. Learning engine reducer、Replay、SM-2、progress、mastery 等任何业务规则实现
2. Recommendation planner、candidate generator、resolver、aggregator、ranker、selector、explanation 等任何业务规则实现
3. Catalog 导入流程、repository、query、sqlc、tx、application/domain/test 层实现
4. 真实数据库 smoke test
5. 自动刷新 Recommendation 物化视图的任务
6. API 层、HTTP 服务、worker、CLI 以外的业务入口

因此，当前项目最准确的定位是：

> **设计文档已经完整，仓库基础层已经落地，后续可以并行实现 Learning engine 和 Recommendation 业务逻辑，但系统还不能执行业务闭环。**

- [cmd/dbtool/main.go](/Users/evan/Downloads/learning-video-recommendation-system/cmd/dbtool/main.go)

当前支持的命令形状是：

```text
dbtool migrate up --module=<catalog|learningengine|recommendation>
dbtool migrate down --module=<...> --steps=<n>
dbtool migrate version --module=<...>
dbtool migrate status --module=<...>
dbtool refresh recommendation
```

当前行为特点：

- 直接读取 `.env` / `DATABASE_URL`
- 模块与 migration 根目录绑定在代码里
- 各模块使用独立 migration tracking table
- Recommendation 的 refresh 只刷新两个物化视图

当前没有：

- 更复杂的命令分组
- 自动刷新任务
- migration plan / drift 检测能力
- 针对 live DB 的安全保护机制

仓库中已经新增 `recommendation` owner migration，包含：

- `recommendation.user_unit_serving_states`
- `recommendation.user_video_serving_states`
- `recommendation.video_recommendation_runs`
- `recommendation.video_recommendation_items`
- `recommendation.v_recommendable_video_units`
- `recommendation.v_unit_video_inventory`
- 对应索引

当前 SQL/query 侧已支持：

- 读取 `learning.user_unit_states`
- 读取 `v_recommendable_video_units`
- 读取 `v_unit_video_inventory`
- 读取 `catalog.video_semantic_spans`
- 读取 `catalog.video_transcript_sentences`
- 读取 `catalog.video_user_states`
- upsert unit/video serving state
- insert run
- insert item
- refresh 物化视图

当前 `v_unit_video_inventory` 已经采用一版仓库内固定的默认规则：

- `strong_video_count` 按 `mention_count >= 2 AND coverage_ratio >= 0.05 AND mapped_span_ratio >= 0.50` 统计
- `supply_grade` 分为 `strong / ok / weak / none`

这一版规则已经写入 migration，因此当前仓库里**确实已经存在 Recommendation 供给感知基础规则的 SQL 表达**，但后续仍允许按 Recommendation owner 边界继续调整。
