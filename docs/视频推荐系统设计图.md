##### 时序图

```mermaid
sequenceDiagram
    autonumber

    actor Client as 上层业务 API / 调用方
    participant Facade as Recommendation 入口
    participant Ctx as 上下文组装器
    participant LS as 学习状态读取器
    participant INV as 内容供给读取器
    participant RVU as 可推荐内容索引读取器
    participant USS as 单元级近期推荐状态读取器
    participant VSS as 视频级近期推荐状态读取器
    participant VUS as 用户视频消费状态读取器（可选）
    participant Planner as 学习需求规划器
    participant Cand as 候选生成器
    participant Exact as 核心精确命中 lane
    participant Bundle as 多单元组合 lane
    participant Future as 受控扩张 lane
    participant Fallback as 质量兜底 lane
    participant Resolver as 证据解析器
    participant Span as 语义证据读取器
    participant Sentence as 句子上下文读取器
    participant Agg as 视频级聚合器
    participant Ranker as 视频排序器
    participant Selector as 最终选择器
    participant Explainer as 推荐解释生成器
    participant Audit as 推荐审计写入器
    participant Serving as 推荐冷却状态管理器

    Client->>Facade: 发起一次视频推荐请求
    Facade->>Ctx: 组装本轮推荐上下文

    Ctx->>LS: 读取当前目标学习状态
    LS-->>Ctx: 返回当前学习阶段、掌握度、下次复习时间、最近表现

    Ctx->>INV: 读取每个学习单元的供给强弱
    INV-->>Ctx: 返回供给概况与强弱等级

    Ctx->>USS: 读取单元级近期推荐状态
    USS-->>Ctx: 返回哪些学习单元最近已被服务

    Ctx->>VSS: 读取视频级近期推荐状态
    VSS-->>Ctx: 返回哪些视频最近已被推荐

    opt 启用用户视频消费状态
        Ctx->>VUS: 读取最近观看、完成、收藏等概况
        VUS-->>Ctx: 返回消费惩罚与去重参考
    end

    Ctx-->>Facade: 返回 RecommendationContext

    Facade->>Planner: 生成本轮学习需求画像
    Planner->>Planner: 解释学习状态，判定紧急度、成熟度、脆弱度
    Planner->>Planner: 将学习单元分桶为当前必须复习、现在可学新内容、近期应顺带覆盖、近未来预热
    Planner->>INV: 结合供给强弱做供给感知
    INV-->>Planner: 返回哪些内容供给弱、哪些内容供给强

    alt 当前主线内容供给偏弱
        Planner->>Planner: 提高组合承载与受控扩张预算，并标记 low_supply 模式
    else 当前主线内容供给正常
        Planner->>Planner: 使用默认预算与配额
    end

    Planner-->>Facade: 返回 DemandBundle

    Facade->>Cand: 生成候选视频集合

    Cand->>Exact: 生成核心精确命中候选
    Exact->>RVU: 读取当前必须复习与高优先级新内容的命中
    RVU-->>Exact: 返回高精度视频-学习单元候选
    Exact-->>Cand: 返回 exact_core 候选

    Cand->>Bundle: 生成多单元组合承载候选
    Bundle->>RVU: 读取当前主线与可顺带覆盖内容的命中
    RVU-->>Bundle: 返回视频-学习单元候选
    Bundle->>Bundle: 按视频聚合，筛出可同时承载多个高价值学习单元的视频
    Bundle-->>Cand: 返回 bundle 候选

    Cand->>Future: 生成受控扩张候选
    Future->>RVU: 读取近期应覆盖内容与近未来预热内容的命中
    RVU-->>Future: 返回补充型候选
    Future-->>Cand: 返回 soft_future 候选

    alt 前三路候选合起来仍明显不足
        Cand->>Fallback: 生成质量兜底候选
        Fallback->>RVU: 读取教育适配度较高且弱相关的视频
        RVU-->>Fallback: 返回 fallback 候选
        Fallback-->>Cand: 返回 quality_fallback 候选
    else 候选量已足够
        Cand->>Cand: 跳过 fallback
    end

    Cand->>Cand: 合并、去重、裁剪各 lane 候选
    Cand-->>Facade: 返回 VideoUnitCandidates

    Facade->>Resolver: 解析候选中的最佳证据窗口

    loop 对每个视频-学习单元候选
        Resolver->>Span: 按代表性证据引用回查语义片段
        Span-->>Resolver: 返回片段时间范围与语义内容
        Resolver->>Sentence: 读取片段所在句子与上下文
        Sentence-->>Resolver: 返回句子窗口与文本上下文
        Resolver->>Resolver: 选择本轮最适合解释与跳转的证据窗口
    end

    Resolver-->>Facade: 返回 ResolvedEvidenceWindows

    Facade->>Agg: 聚合为视频级候选
    Agg->>Agg: 同视频同单元去重
    Agg->>Agg: 统计该视频覆盖了多少当前主线与次级需求
    Agg->>Agg: 构造覆盖强度、组合价值、教育适配度、未来价值等视频级特征
    Agg-->>Facade: 返回 VideoCandidates

    Facade->>Ranker: 对视频级候选做综合打分
    Ranker->>Ranker: 计算学习需求覆盖分
    Ranker->>Ranker: 计算内容承载与教学性分
    Ranker->>Ranker: 扣除近期重复、已看、过载等惩罚
    Ranker-->>Facade: 返回 RankedVideos

    Facade->>Selector: 选择最终视频列表
    Selector->>Selector: 判断 normal、low_supply、extreme_sparse 三种模式
    Selector->>Selector: Phase A 先保当前主线覆盖
    Selector->>Selector: Phase B 按边际收益继续填充

    alt 仍不足且允许少量兜底
        Selector->>Selector: 接受少量 fallback 视频补位
    else 仍不足但兜底也不合理
        Selector->>Selector: 允许 under-fill，避免明显弱相关内容进入
    end

    Selector-->>Facade: 返回 SelectedVideos

    Facade->>Explainer: 为最终视频生成推荐解释

    loop 对每条最终视频
        Explainer->>Explainer: 生成 reason codes
        Explainer->>Explainer: 生成覆盖内容与推荐原因
        Explainer->>Explainer: 生成可跳转的证据位置
    end

    Explainer-->>Facade: 返回 FinalVideoList

    Note over LS,Sentence: Recommendation 只读取 Learning engine 与 Catalog 的输出，不回写它们
    Note over Audit,Serving: 审计写入与推荐冷却状态更新属于 Recommendation 自己的写路径

    Facade->>Audit: 写入本轮推荐运行记录与最终视频明细
    Audit-->>Facade: 审计写入完成

    Facade->>Serving: 更新学习单元级与视频级推荐冷却
    Serving-->>Facade: 冷却状态更新完成

    Facade-->>Client: 返回最终视频列表、推荐原因、证据位置、run_id、selector_mode、underfilled
```
