# 全局约束（执行所有任务前先读）

| 约束项 | 详细说明 |
| --- | --- |
| 模块边界 | 最终必须拆成 `internal/learningengine` 与 `internal/recommendation` 两个平级模块。 |
| 无兼容 | 不保留 `internal/recommendation/scheduler` 作为兼容壳，不保留旧 usecase/旧 repository 接口的转发实现。 |
| 数据 owner | `learning.unit_learning_events` 与 `learning.user_unit_states` 只由 Learning engine 维护；Recommendation 不允许写这两张表。 |
| 推荐 owner | `recommendation.user_unit_serving_states`、`recommendation.scheduler_runs`、`recommendation.scheduler_run_items` 只由 Recommendation 维护。 |

| Replay 边界 | MVP replay 只属于 Learning engine，只支持 full replay；Recommendation 不允许保留 replay 能力。 |
| 持久化方案 | 继续使用 `sqlc + pgx/v5 + PostgreSQL 直连`，不引入 ORM。 |
| 规则落点 | Learning engine 的状态规则保留在 Learning engine 的 `domain` 内；Recommendation 的 quota/scorer/assembler 保留在 Recommendation 的 `domain` 内。 |

| 测试要求 | 每个关键拆分点都必须有编译验收、领域单测、集成测试或迁移验证。 |
